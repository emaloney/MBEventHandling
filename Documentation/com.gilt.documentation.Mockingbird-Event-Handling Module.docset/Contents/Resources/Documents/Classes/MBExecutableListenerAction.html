Overview

The base class for listener action implementations that can be directly executed.

Presents a higher-level API for handling events, used by actions that do not require knowledge about the listener handling the event and the action’s container.

Subclasses must implement the executeForEvent: method to handle events.

Subclasses should not override eventReceived:byListener:container:.

Action execution

– executeForEvent:

Implemented by subclasses to perform the action when an event is received.

- (void)executeForEvent:(nullable NSNotification *)event

Parameters

event

the NSNotification that triggered the event listener. This parameter may be nil if the action is not being executed as the result of an event trigger (in other words, when the listener action is being invoked manually).

Discussion

This method is called by the default implementation of the method eventReceived:byListener:container:.

– validateAndExecuteForEvent:

Attempts to validate the receiver’s data model and, if successful, executes the event using the passed-in notification.

- (BOOL)validateAndExecuteForEvent:(nullable NSNotification *)event

Parameters

event

the NSNotification that triggered the event listener. This parameter may be nil if the action is not being executed as the result of an event trigger (in other words, when the listener action is being invoked manually).

Return Value

YES if the data model validated and the event was executed; NO if the event did not execute because the data model wasn’t valid.
