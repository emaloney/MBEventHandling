MBListenerAction Class Reference

Inherits from MBEventHandler : MBDataModel
Declared in MBEventListener.h

Overview

Provides a low-level API for performing actions in response to events received by listeners.

Subclasses must implement the eventReceived:byListener:container: method to handle events.

Action implementations should usually subclass MBExecutableListenerAction, which provides a higher-level API for executing actions.

Configuring the action's behavior

– acceptsTextContent

Indicates whether the action accepts text content within its XML tags.

- (BOOL)acceptsTextContent

Return Value

YES if text content is accepted; NO otherwise.

Discussion

Default implementation returns NO.

– acceptsArbitraryAttributes

Indicates whether the action accepts an arbitrary set of attributes. Default implementation returns NO. If YES, attribute validation will not occur and arbitrary events will be accepted without issuing console warnings.

- (BOOL)acceptsArbitraryAttributes

Return Value

YES if the listener accepts arbitrary attributes; NO otherwise.

MBEventListener.h

– ignoresNonstandardAttributes

Indicates whether the action ignores nonstandard attributes (i.e., those other than “if”, “afterDelay” and “warnOnOverwrite”.)

- (BOOL)ignoresNonstandardAttributes

Return Value

YES if the listener ignores nonstandard attributes; NO otherwise.

Discussion

Default implementation returns NO. If YES, attribute validation will fail if any attributes appear other than the standard action attributes listed above.
